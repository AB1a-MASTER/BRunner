BYPASS EXECUTION

Files:
- `BRunner/core/stepBypass.js`
- `BRunner/core/variableRegistry.js`
- `BRunner/background.js`

Modes:
- Enabled: execute normally.
- Bypassed: always skip while preserving graph connections.
- Conditional bypass: evaluate `skipWhen` immediately before the node.

Accepted conditional values:
- true/false
- yes/no
- on/off
- 1/0

Static bypass is checked before other node expressions, so unused missing
variables in a disabled node cannot fail the workflow. Missing or ambiguous
conditional values fail with distinct diagnostics. The legacy `disabled: true`
flag remains supported.

DEFERRED VISIBILITY/PRECONDITION REQUIREMENT

The docs now require registry-level capability metadata for nodes that depend on
actual on-screen browser state. Future work must identify and enforce whether a
node requires:
- foreground/active tab;
- focused browser window;
- target visible and unobscured in the viewport;
- target scrolled into view;
- hardware/pointer interaction.

Hover Mouse is the primary example. The runtime must activate/focus/present the
target before dispatch so the node does not fail because the tab or element was
off-screen. Screenshot, hardware keyboard/mouse, drag, and similar nodes require
the same audit. Background-safe HTTP/data operations must not force foreground
activation.
